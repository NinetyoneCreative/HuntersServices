## 1. What this site is

A fast, static, SEO-optimized marketing site for Hunters Services Inc. — 16 pages plus a
thank-you page and a custom 404. No database, no build server required; it's plain HTML/CSS/JS
that can be hosted anywhere (Netlify recommended).

### Page inventory
| Page | File | Purpose |
|---|---|---|
| Home | `index.html` | Conversion hero, trust strip, 7-service grid, story, reviews, areas, estimate form, CTA |
| Services index | `services.html` | Grid of all 7 services |
| About | `about.html` | Company story, values, stats, mission, reviews |
| Contact | `contact.html` | Free-estimate form, direct contact, map, service areas |
| 7 service pages | `services/{pest-control,termites,bed-bugs,rodents,wildlife,weed-abatement,tree-services}.html` | Localized hero, intro, IPM approach, process, FAQs, sidebar quote form, reviews |

| Thank-you | `thank-you.html` | Form confirmation + conversion landing (noindex) |
| 404 | `404.html` | Friendly not-found page (noindex) |

## 2. Build system — Node.js / Eleventy

The site is now generated by a **Node.js static-site build using Eleventy (11ty)**, living in
**`hunters-services-11ty/`**. This replaces the original (never-provided) Python generator and gives
the same DRY benefit: all shared chrome — `<head>` + JSON-LD, top bar, nav, footer, floating call
button, scripts — is defined **once** in a single layout, and each page only holds its unique body.

```
hunters-services-11ty/
├── package.json            # scripts: dev / build / clean
├── .eleventy.js            # config: passthrough copy, watch targets, output = _site
├── src/
│   ├── _includes/layouts/base.njk   # the ONE shared layout (head, nav, footer, schema)
│   ├── index.html  services.html  about.html  contact.html  thank-you.html  404.html
│   ├── services/*.html  (7)         # each = front matter + unique body only
│   ├── areas/*.html     (5)
│   ├── assets/  css/  js/           # copied verbatim to output
│   └── static/                      # robots.txt, sitemap.xml, netlify.toml, logo.png, og.png → output root
└── _site/                  # BUILD OUTPUT — this is what you deploy
```

Each page's `.html` file starts with YAML front matter (`title`, `description`, `canonical`, `nav`
for the active menu item, and any page-specific `extraSchema` JSON-LD) followed by the page body.
The body HTML is passed through verbatim (`htmlTemplateEngine: false`), so it's plain HTML — no
template syntax to learn for content edits.

### Local development (live reload)
```powershell
cd hunters-services-11ty
npm install          # first time only
npm run dev          # serves http://localhost:8123 with hot reload — edit src/*, browser refreshes
```
`npm run dev` watches `src/` (pages, layout, CSS, JS, assets) and rebuilds + live-reloads the
browser on every save. **Leave it running while you work.**

### Production build
```powershell
cd hunters-services-11ty
npm run build        # writes the deployable site to _site/
```
Deploy the **`_site/`** folder (or the `hunters-services-deploy.zip`, which is built from `_site/`).

> The earlier flat-HTML version still exists at `hunters-services-site/` as a reference snapshot;
> `hunters-services-11ty/` is now the source of truth. `_tools/` holds the one-off migration and
> image-optimization scripts (not part of the deploy).

### To change shared elements vs. page content
- **Header, nav, footer, `<head>`, business schema, phone/email/address:** edit `src/_includes/layouts/base.njk` once.
- **A page's copy:** edit that page's file in `src/` (e.g. `src/services/termites.html`).
- **Form behavior:** `src/js/main.js`. **Styles:** `src/css/styles.css`.

## 3. What was completed (open items from the brief)

### ✅ Lead form → real endpoint (Netlify Forms)
- All **15 forms** wired to **Netlify Forms** (`name="estimate"`, `method="POST"`,
  `data-netlify="true"`, `action="/thank-you.html"`).
- **Honeypot spam protection:** hidden `bot-field` + `data-netlify-honeypot` (Netlify also runs
  Akismet automatically). Visually hidden and skipped by screen readers.
- **Validation:** native HTML5 (required + email type) plus a JS safety net in `main.js`
  (name length, phone digit count, email pattern).
- **Basic rate limit:** rapid re-submits within 4s are blocked (client-side), and the submit
  button is disabled while sending to prevent doubles.
- **Success / error states:** inline success confirmation on submit; a friendly error message
  with the phone number if the network call fails.
- **Conversion event:** on successful submit, `main.js` pushes a `generate_lead` event to
  `window.dataLayer` (GTM-ready) and calls `gtag('event','generate_lead', …)` and
  `fbq('track','Lead')` if those are present. The thank-you page also fires it for the no-JS path.
- **Source tracking:** each form sends a hidden `source-page` field so you can see which page the
  lead came from in the Netlify dashboard.
- **No-JS fallback:** with JavaScript off, the form still posts natively to Netlify and lands on
  `/thank-you.html`.

**⚠️ One manual step after first deploy — set the notification email:**
Netlify form-submission notifications are a dashboard setting (not in code). After the site is
live: **Netlify → Site → Forms → Settings & usage → Form notifications → Add notification →
Email notification → send to `marketingcoordinator@huntersservices.net`** (form: *estimate*).
Until that's set, submissions are still captured in the dashboard — you just won't get the email.

### ✅ Service images → local optimized assets (no Wix hotlinks)
- All **23 image references** that previously hotlinked `static.wixstatic.com` now point to local
  files in `assets/services/`. **Zero external image dependencies remain.**
- The client's own photos were downloaded from the Wix CDN and re-encoded with `sharp` into
  **WebP (primary) + JPEG (fallback)**, in two sizes each (`-card` 760w, `-wide` 1100w), served
  via `<picture>`.
- **Graceful fallback:** every service-card image sits on the navy gradient + line-icon; if an
  image ever fails to load, `onerror` hides it and the gradient + icon show through.
- Brand hero optimized too: `technician.png` **1.26 MB → 66 KB** WebP (the LCP image, now
  preloaded with `fetchpriority="high"`); `logo.png` **332 KB → 92 KB**.

### ✅ Verification (local)
- Ran a local server and screenshotted **home, a service page, and a city page at desktop
  (1320px) and mobile (390px)** — layouts confirmed correct.
- **All 933 local links/asset references resolve** (automated check, `_tools/linkcheck.js`).
- **All JSON-LD blocks parse as valid** (`_tools/schemacheck.js`): LocalBusiness/PestControlService
  on every page, Service + FAQPage on service pages, localized business + FAQPage on city pages,
  aggregateRating 4.3 / 708.
- Form success flow, validation blocking, and the conversion `dataLayer` push were verified in the

### ✅ SEO / infrastructure added

## 4. How to deploy to Netlify

The site is a **static publish — no build command**. The publish directory is the site folder
itself (`netlify.toml` sets `publish = "."`).

### Option A — Drag & drop (no account access needed from us)
1. Go to https://app.netlify.com/drop
2. Drag **`hunters-services-deploy.zip`** onto the page (or drag the `hunters-services-site` folder).
3. Netlify gives you a live URL like `https://random-name-123.netlify.app`.
4. Rename it under **Site configuration → Change site name** (e.g. `hunters-services`).
5. Do the **form notification** step from §3 and the **custom domain** step from §5.

### Option B — Netlify CLI (repeatable)
```powershell
cd hunters-services-11ty
npm run build                              # generates _site/
npx netlify-cli login                      # one-time, opens a browser
npx netlify-cli deploy --dir "_site" --prod
```
The CLI prints the production URL. (A Netlify **Personal Access Token** set as
`NETLIFY_AUTH_TOKEN` lets this run non-interactively in CI.)

### Option C — Connect the Git repo (recommended long-term)
Push `hunters-services-11ty/` to GitHub, then **Netlify → Add new site → Import from Git**:
- **Base directory:** `hunters-services-11ty`
- **Build command:** `npm run build`
- **Publish directory:** `hunters-services-11ty/_site`

Netlify then rebuilds automatically on every push.

## 5. Pointing the custom domain (hunterspestcontrol.com)

The canonical URLs use **www.hunterspestcontrol.com** (matches the existing `<link rel="canonical">`
tags and the apex→www redirect in `netlify.toml`).

1. In Netlify: **Site configuration → Domain management → Add a domain →** enter
   `hunterspestcontrol.com`. Add `www.hunterspestcontrol.com` as the **primary** domain.
2. **At your DNS provider** (where the domain is registered — likely currently pointed at Wix):
   - **Recommended (Netlify DNS):** change the domain's **nameservers** to the four Netlify
     nameservers Netlify shows you. Netlify then manages records and auto-provisions HTTPS.
   - **Or keep your DNS host and add records:**
     - `www` → **CNAME** → `your-site-name.netlify.app`
     - apex `@` → Netlify's **A record / ALIAS** as shown in the Netlify domain panel
       (Netlify's load-balancer IP, or an ALIAS/ANAME to the netlify subdomain).
3. **Remove the old Wix domain pointing** for these records so they don't conflict.
4. Wait for DNS to propagate (minutes–48h). Netlify auto-issues a **Let's Encrypt SSL**
   certificate once DNS resolves — confirm **HTTPS** is enabled and **Force HTTPS** is on.
5. Verify `https://hunterspestcontrol.com` 301-redirects to `https://www.hunterspestcontrol.com`.

> ⚠️ Email (`@huntersservices.com` / `@huntersservices.net`) uses **MX records** — do **not**
> change those when you repoint the website, or email will break.

## 6. How to make common edits

- **Phone, email, address, hours, rating** appear in the topbar/footer/JSON-LD of each page.
  They're consistent across files — use find-and-replace across `hunters-services-site/` if a
  business fact changes. Phone link format: `tel:+15303428950`.
- **Form behavior** (validation, success text, conversion event): `js/main.js` only.
- **Colors, fonts, spacing, components:** `css/styles.css` (CSS custom properties at the top:
  `--navy`, `--steel`, `--blue`, `--green`, `--orange`, etc.).
- **Service/area copy:** edit the relevant `services/*.html` or `areas/*.html` directly.
- **Replacing a service image:** drop a new photo in `assets/services/` as
  `{slug}-card.webp/.jpg` and `{slug}-wide.webp/.jpg` (or re-run `_tools/optimize.js` with a new
  source). Sizes: card = 760px wide, wide = 1100px wide.
- **Analytics:** paste your GA4/GTM snippet into the `<head>` of each page (or, if you restore a
  generator/templating, into the shared head). The `generate_lead` conversion event will then flow
  automatically — no JS changes needed.
